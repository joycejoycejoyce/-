想让多个元素 汇聚在同一行的小心愿

大家遇到的问题

大家在排列元素们在同一行的时候都会情不自禁的用 width: 50%; 设置给两个子元素。这个时候讨厌的事情就发生啦。 你会惊讶的发现，“根本就没办法在同一行啊！为什么还是两行呢”。

邪恶的 html 空白折叠

其实问题就是html 的换行，tab, space 都会被解析为一个space. 空白折叠就是如果我们hit tab\space\change line 很多次 browser 也只会把它看作是一个space.浏览器在内部解析时 其实是把这个空格解析成一个& nbsp 符号。
IDE:space space space ---> BROSWER:& nbsp;

破除诅咒的方法以及方法比较

有两个方法可以解决这个问题。
  • 把宽度设置为 width: (100%/elementNumber-1)
    这个方法应该是最常用也最intuitive的了。而且没什么坏处。
  • 可以在父元素的css中将font-size:0px
    既然多出来的是我们的一个 "& nbsp" 符号。那么将父元素的字体大小设为0px就相当于是 不去显示这个字了。
    这个方法也不赖，就是要一定一定记得 在子元素中把font-size要手动重新设回来，要不文字就都会消失的

关于css 属性选择器

解决了什么问题

有时我们希望能动态的来展示一个元素，而并不是永久的移除它。打一个比方，比如说用户sign in 与 sign out 的时候， 我们希望根据 boolean signIn 来决定在heading中出现什么。
用js 的dom操作来把html页面中的元素移进移出一定是一个坏的方法。dom tree 每次都要在改变时重新生成，对网页的performance 是很大的负担和浪费。那么这个时候，我们可以把希望寄托在css身上。css是专门管展示的地方。

  • opacity: 0
  • outline: none
  • visibility: none
