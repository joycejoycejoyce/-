IO 流

IO流主管的是数据的读取与写入。比如说我们常用的文件下载，就是一个数据读取。Java通过在后台 从hard disk中读取写入data来处理data的改变。InputStream 就是从hard disk 中读取数据。 OutputStream 就是把数据从memory中写到 hard disk 上。

OutputStream

OutputStream 是一个abstract class. 它自身并不能创建instance. 但是它的子类 FileOutputStream 可以。我会重点谈FileOutputStream.

syntax
在建立新的instance 时，有三件事情发生了。
  1. 建立了fileOutputStream instance
  2. 指定了文件所在的路径，如果路径下没有相应名字的file,Java会自动创建一个
  3. 将instance －> file
FileOutputSteam 写入的三种方式
这里java 用了多态的概念出了三个name相同但是param 不同的三个method
  • write(int i)
  • write(byte[])
  • write(byte[], startIndex, length)
写入文件时的两个小问题
  • 如何换行
    用“\n”来表示换行。不过有一点要注意的是在不同的操作系统下，对于"\n"的解析是不一样的
    OSparse
    Linux"\n"
    Windows"\r\n"
    MacOS"\r"
  • 如何续写文件
    在创建instance 时加上第二个param设为true. 第二个param 是在问programmer是否要接着续写file。
IO流处理异常－－finally块
finally block 的作用为执行清除操作。在finally 块中常做的操作是释放资源。我们需要finally block 的原因是 它是一个只要JVM不退出就一定会执行的block。
